# Cambio de método — Prototipo simulado Gamora Bot

## Estado

Documento rector de cambio de método.

## Contexto

Gamora Bot cuenta con una documentación estratégica y funcional avanzada, así como con antecedentes de desarrollo técnico hasta MVP 2.5.1.

Sin embargo, antes de continuar hacia MVP 3 — WhatsApp mock controlado, se decide pausar la ejecución técnica y cambiar temporalmente el enfoque hacia la construcción de un prototipo simulado, estático y sin código.

## Decisión

A partir de esta etapa, el proyecto se trabajará exclusivamente entre Luis Felipe y ChatGPT.

No se involucrará Codex en ninguna fase de esta etapa.

El objetivo no será desarrollar software funcional, sino diseñar una experiencia simulada que permita explicar, mostrar y validar la idea de producto con usuarios reales o potenciales.

## Alcance de esta etapa

Esta etapa incluirá:

- Reconstrucción del estado maestro del proyecto.
- Definición del caso demo principal.
- Diseño de narrativa.
- Diseño de flujo.
- Diseño de mensajes simulados.
- Diseño de pantallas base.
- Wireframes de baja fidelidad.
- Storyboard.
- Prototipo estático.
- Guion de validación con usuarios.
- Criterios para interpretar feedback.

## Fuera de alcance

Durante esta etapa no se desarrollará:

- Código.
- Backend.
- Base de datos.
- Integración real con WhatsApp.
- Integración real con OpenAI.
- Webhook.
- Automatizaciones reales.
- MVP 3.
- Aplicación funcional.

## Principio rector

La prioridad será validar claridad, utilidad, fricción percibida y deseo de uso antes de invertir más esfuerzo técnico.

La secuencia de trabajo será:

1. Idea clara.
2. Narrativa clara.
3. Flujo claro.
4. Textos claros.
5. Pantallas claras.
6. Validación.
7. Diseño visual.
8. Decisión sobre siguiente etapa técnica.

## Criterio de éxito

Esta etapa será exitosa si un usuario puede entender en pocos minutos que Gamora Bot le ayuda a convertir conversaciones operativas de WhatsApp en compromisos trazables, con evidencia, revisión y cierre.

## Regla crítica

No se avanzará a MVP 3 hasta que el prototipo simulado haya sido estructurado, revisado y usado para validar la idea de producto.
