# Dirección Visual Inicial — Gamora Bot

## Estado

Documento rector inicial de dirección visual para el prototipo simulado de Gamora Bot.

## Propósito

Este documento define la primera dirección visual de Gamora Bot antes de construir el prototipo visual estático.

Su objetivo es traducir toda la estrategia, narrativa, flujo y experiencia ya aprobada en una línea visual clara, profesional, coherente y validable.

Este documento no es todavía un UI Kit completo.

No es diseño final.

No es prototipo visual.

No es branding definitivo.

Es la guía visual inicial que debe evitar que el prototipo se vea improvisado, genérico, infantil, demasiado corporativo o desconectado de la tesis del producto.

---

# Principio rector del producto

**WhatsApp opera. Web/PWA estructura, coordina y visualiza. Gamora mantiene el hilo entre ambas.**

Toda decisión visual debe reforzar este principio.

Si una decisión visual complica este principio, se descarta.

---

# Frase central del producto

**Tus pendientes se atienden en WhatsApp, pero no se pierden en WhatsApp.**

Esta frase debe orientar el diseño visual.

La interfaz debe transmitir que Gamora Bot no reemplaza el hábito natural del usuario, sino que lo convierte en control operativo.

---

# Objetivo visual del prototipo

El prototipo visual debe lograr que un usuario entienda rápidamente:

1. Que Gamora Bot vive naturalmente en WhatsApp.
    
2. Que no obliga al usuario operativo a aprender una plataforma pesada.
    
3. Que la Web/PWA no está de más.
    
4. Que Web/PWA es la capa que ordena, coordina, visualiza e historiza.
    
5. Que ambas interfaces forman una sola experiencia.
    
6. Que la evidencia, corrección y cierre quedan claros.
    
7. Que la experiencia se siente ligera, confiable y profesional.
    

---

# Tesis visual

Gamora Bot debe verse como una capa de control inteligente que ordena la operación cotidiana sin interrumpirla.

No debe sentirse como:

- una app de tareas genérica,
    
- un CRM,
    
- un ERP,
    
- una plataforma corporativa pesada,
    
- un chatbot casual,
    
- un sistema de vigilancia,
    
- una herramienta infantil,
    
- una demo técnica.
    

Debe sentirse como:

- asistente operativo,
    
- capa de coordinación,
    
- sistema de compromisos,
    
- memoria visual del trabajo,
    
- puente natural entre WhatsApp y control estructurado.
    

---

# Concepto visual rector

## Nombre del concepto

**Operación Conversacional Estructurada**

## Definición

La experiencia visual debe mostrar que una conversación puede convertirse en estructura sin perder naturalidad.

WhatsApp representa el movimiento rápido de la operación.

Web/PWA representa el orden, la memoria y el control.

Gamora Bot representa el hilo que conecta ambas.

## Traducción visual

- WhatsApp: ligero, conversacional, directo.
    
- Gamora Bot: claro, confiable, guía sin invadir.
    
- Web/PWA: estructurada, limpia, visualmente ordenada.
    
- Transiciones: suaves, contextuales, sin sensación de cambio brusco.

# Posicionamiento visual

Gamora Bot debe ubicarse visualmente entre tres mundos:

## 1. Familiaridad de WhatsApp

Debe sentirse cercano al entorno donde el usuario ya opera.

Pero no debe copiar de forma literal ni depender visualmente de WhatsApp como marca.

La interfaz tipo WhatsApp debe verse reconocible, pero propia.

## 2. Claridad de producto SaaS moderno

La Web/PWA debe verse como una herramienta profesional, limpia y confiable.

Debe dar sensación de control sin sentirse complicada.

## 3. Inteligencia operativa

Gamora Bot debe verse como un asistente que estructura y coordina.

No como un personaje, no como un robot caricaturesco y no como una IA futurista exagerada.

# Dirección estética recomendada

## Estilo general

**Minimalismo operativo cálido.**

Esto significa:

- pocas distracciones,
    
- buena jerarquía,
    
- espacios amplios,
    
- textos claros,
    
- componentes limpios,
    
- estados visibles,
    
- énfasis en acciones,
    
- sensación humana,
    
- detalles visuales sutiles.
    

## No usar

- fondos oscuros excesivos,
    
- neones futuristas,
    
- gradientes agresivos,
    
- saturación de íconos,
    
- dashboards densos,
    
- tarjetas demasiado cargadas,
    
- burbujas de chat interminables,
    
- estética de soporte técnico,
    
- estética de auditoría,
    
- estética de task manager.

# Uso estratégico del color

## WhatsApp / conversación

Debe usar una base clara y familiar, con burbujas suaves.

No debe abusar del verde.

El verde debe marcar acciones y confirmaciones, no pintar toda la experiencia.

## Web/PWA

Debe usar una estructura más sobria:

- fondos claros,
    
- tarjetas blancas,
    
- acentos verde operativo,
    
- detalles azul petróleo,
    
- badges de estado claros.
    

## Estados

Los estados deben ser visualmente fáciles de distinguir, pero sin parecer sistema de auditoría.

Ejemplos:

- Borrador: gris.
    
- Nuevo: azul grisáceo.
    
- Aceptado: azul.
    
- Evidencia enviada: azul petróleo.
    
- Necesita corrección: ámbar.
    
- Corregido: verde suave.
    
- Cerrado: verde sólido.

# Tipografía sugerida

## Estilo tipográfico

La tipografía debe ser:

- moderna,
    
- legible,
    
- humana,
    
- limpia,
    
- sin exceso de personalidad.
    

## Recomendación

Usar una sans serif contemporánea.

Opciones conceptuales:

- Inter.
    
- Geist.
    
- Manrope.
    
- SF Pro como referencia estética.
    
- Segoe UI como referencia Windows-friendly.
    

## Regla

No usar tipografías decorativas.

No usar tipografías demasiado tecnológicas.

No usar tipografías que parezcan de app financiera compleja.

La letra debe desaparecer para que el flujo se entienda.

---

# Forma y componentes

## Bordes

Usar bordes redondeados moderados.

Recomendación:

- tarjetas: 16px,
    
- botones: 12px,
    
- burbujas de chat: 16px a 20px,
    
- badges: 999px.
    

La experiencia debe sentirse amable, pero no infantil.

## Sombras

Usar sombras muy suaves.

La interfaz debe sentirse ligera, no flotante en exceso.

## Espaciado

Usar mucho espacio para respirar.

Gamora Bot debe sentirse simple.

Si una pantalla parece llena, probablemente está mal.

---

# Iconografía

## Estilo recomendado

Íconos lineales, simples y claros.

No usar íconos muy técnicos.

No usar íconos caricaturescos.

## Íconos útiles

- compromiso,
    
- foto/evidencia,
    
- reloj/fecha,
    
- responsable/persona,
    
- ubicación,
    
- revisión,
    
- corrección,
    
- cierre,
    
- historial,
    
- WhatsApp/chat,
    
- Web/PWA/tablero.
    

## Regla

El ícono debe ayudar a escanear, no decorar.

# Dirección visual para WhatsApp simulado

## Objetivo

Mostrar que el usuario opera desde un entorno conversacional familiar.

## Reglas

- Usar burbujas claras.
    
- Mensajes breves.
    
- Acciones con botones visibles.
    
- Evitar conversaciones largas de pantalla completa.
    
- Resaltar los resúmenes estructurados de Gamora Bot.
    
- Mostrar adjuntos de evidencia de forma simple.
    
- Mostrar estados sin saturar.
    

## Mensajes de Gamora Bot

Los mensajes de Gamora deben verse diferentes a los mensajes normales.

Deben sentirse como tarjetas conversacionales estructuradas.

Ejemplo visual:

- encabezado breve,
    
- campos detectados,
    
- botón principal,
    
- acciones secundarias.
    

## Mensajes de usuario

Los mensajes de Luisito y Panchito deben sentirse naturales.

No deben parecer formularios.

## Botones dentro de WhatsApp

Deben ser claros:

- Crear compromiso.
    
- Aceptar compromiso.
    
- Enviar evidencia.
    
- Pedir corrección.
    
- Aprobar y cerrar.
    
- Confirmar cierre.
    
- Abrir en Web/PWA.
    

## Regla crítica

La conversación no debe parecer un chatbot infinito.

Debe parecer un flujo operativo guiado.

---

# Dirección visual para Web/PWA

## Objetivo

Mostrar que la Web/PWA es el centro visual de control, no una app obligatoria para operar.

## Sensación deseada

La Web/PWA debe sentirse como:

- tablero limpio,
    
- ficha de compromiso,
    
- historial claro,
    
- evidencia ordenada,
    
- control visual.
    

## Pantallas Web/PWA clave

Solo tres apariciones son indispensables en el prototipo visual:

1. Compromiso creado.
    
2. Revisión ampliada de evidencia.
    
3. Historial final.
    

## Componentes Web/PWA mínimos

### 1. Tarjeta de compromiso

Debe mostrar:

- nombre del compromiso,
    
- estado,
    
- responsable,
    
- fecha,
    
- evidencia esperada,
    
- último evento.
    

### 2. Vista de evidencia

Debe mostrar:

- foto,
    
- cantidad reportada,
    
- responsable,
    
- acciones,
    
- historial parcial.
    

### 3. Historial

Debe mostrar:

- eventos principales,
    
- horas,
    
- actor,
    
- estado final,
    
- evidencia final.
    

### 4. Botón de continuidad

Debe aparecer claramente:

- Volver a WhatsApp.
    
- Abrir conversación en WhatsApp.
    

## Regla crítica

La Web/PWA no debe verse como dashboard lleno de métricas.

Primero debe verse como memoria operativa del compromiso.

---

# Diseño de la conexión entre interfaces

Esta es la parte más importante del prototipo visual.

## Objetivo

Que WhatsApp y Web/PWA se sientan como dos vistas del mismo flujo.

## Cómo mostrarlo

### Desde WhatsApp

Botón:

**Abrir en Web/PWA**

Debe aparecer en momentos donde aporta valor:

- después de crear compromiso,
    
- al revisar evidencia,
    
- al cierre.
    

### Desde Web/PWA

Botón:

**Volver a WhatsApp**

o

**Abrir conversación en WhatsApp**

Debe aparecer en:

- detalle del compromiso,
    
- revisión de evidencia,
    
- historial final.
    

## Regla

No se debe sentir como “abrir otra aplicación”.

Debe sentirse como “ver este mismo compromiso de otra forma”.

# Jerarquía visual por actor

Luisito necesita ver:

- control,
    
- estado,
    
- evidencia,
    
- corrección,
    
- cierre,
    
- historial.

## Panchito

Panchito necesita ver:

- qué hacer,
    
- para cuándo,
    
- qué evidencia mandar,
    
- qué corregir,
    
- confirmación de cierre.
    

Su experiencia debe ser mínima.

## Gamora Bot

Gamora Bot necesita verse como:

- guía,
    
- organizador,
    
- estructurador,
    
- coordinador,
    
- no como autoridad punitiva.

# Estados visuales

## Borrador

Uso:

Cuando Gamora estructuró una instrucción, pero Luisito aún no confirma.

Estilo:

- badge gris,
    
- texto suave,
    
- “Pendiente de confirmación”.
    

## Nuevo

Uso:

Compromiso creado y enviado.

Estilo:

- badge azul grisáceo.
    

## Aceptado

Uso:

Panchito confirmó atención.

Estilo:

- badge azul o verde suave.
    

## Evidencia enviada

Uso:

Panchito envió evidencia para revisión.

Estilo:

- badge azul petróleo.
    

## Necesita corrección

Uso:

Luisito pidió ajuste.

Estilo:

- badge ámbar.
    

Lenguaje visual:

Debe llamar la atención sin verse como error grave.

## Corregido

Uso:

Panchito envió evidencia corregida.

Estilo:

- badge verde suave.
    

## Pendiente de cierre

Uso:

Luisito está por confirmar cierre.

Estilo:

- badge azul/ámbar suave.
    

## Cerrado

Uso:

Compromiso aprobado y terminado.

Estilo:

- badge verde sólido.
    

---

# Tono visual de la corrección

La corrección es uno de los momentos más delicados.

Debe sentirse como:

- ajuste,
    
- guía,
    
- precisión,
    
- mejora.
    

No debe sentirse como:

- rechazo,
    
- castigo,
    
- incumplimiento,
    
- regaño,
    
- falla.
    

## Recomendación visual

Usar ámbar suave, no rojo.

Usar textos como:

- “Se necesita una corrección.”
    
- “Falta una foto más abierta.”
    
- “Panchito ya sabe qué reenviar.”
    

No usar:

- “Evidencia rechazada.”
    
- “Incumplimiento.”
    
- “Error.”
    
- “No válido.”
    

---

# Tono visual del cierre

El cierre debe ser el momento de mayor satisfacción.

Debe sentirse:

- claro,
    
- final,
    
- ordenado,
    
- confiable.
    

## Elementos recomendados

- estado “Cerrado” visible,
    
- resultado registrado,
    
- evidencia final,
    
- historial,
    
- frase de alivio.
    

Frase sugerida:

**Ya no tienes que dar seguimiento manual a este pendiente.**

---

# Marca verbal dentro de la interfaz

Gamora Bot debe hablar con frases cortas, operativas y humanas.

## Estilo de mensajes

Correcto:

- “Detecté este compromiso.”
    
- “¿Lo creo así?”
    
- “Panchito ya aceptó.”
    
- “Envié la evidencia a Luisito para revisión.”
    
- “Se necesita una corrección.”
    
- “¿Confirmas el cierre?”
    

Evitar:

- “He procedido a ejecutar el workflow.”
    
- “Se ha generado una entidad operativa.”
    
- “La evidencia ha sido rechazada.”
    
- “El proceso ha sido liberado.”
    
- “El usuario ejecutor ha completado la actividad.”

# Identidad de Gamora Bot

## Logo / símbolo

Para el prototipo inicial, no es indispensable cerrar logo final.

Se recomienda usar una marca simple:

**Gamora Bot**

Con un símbolo abstracto opcional que represente:

- hilo,
    
- flujo,
    
- check,
    
- conversación,
    
- estructura.
    

## Evitar

- estética de robot,
    
- estética de superhéroe,
    
- referencia visual a personajes existentes,
    
- mascota caricaturesca,
    
- exceso de IA futurista.
    

## Posible idea simbólica

Un símbolo que combine:

- burbuja de conversación,
    
- línea de flujo,
    
- check/cierre,
    
- nodo de conexión.
    

Concepto:

**del mensaje al compromiso cerrado.**

# Layout general recomendado

## Para escenas WhatsApp

Usar formato vertical tipo móvil.

Debe parecer capturable en una pantalla de teléfono.

## Para escenas Web/PWA

Usar formato dashboard ligero.

Puede mostrarse en escritorio o tablet.

Para la primera demo, se recomienda mostrar Web/PWA como panel amplio y limpio, no como app móvil.

Esto ayuda a distinguir:

- WhatsApp = operación móvil.
    
- Web/PWA = control visual amplio.
    

## Composición de demo

En escenas clave, se puede mostrar:

- móvil WhatsApp a la izquierda,
    
- panel Web/PWA a la derecha,
    
- línea o indicador de sincronización entre ambos.
    

Esto puede reforzar la idea de doble interfaz conectada.

# Estilo de presentación del prototipo

## Recomendación

El prototipo visual debe poder presentarse como una secuencia de slides o pantallas.

Cada escena debe incluir:

- título breve,
    
- interfaz simulada,
    
- mensaje clave,

- poner muchos elementos decorativos,
    
- saturar con métricas,

# Dirección para la primera versión visual

## Versión visual V1

Debe ser:

- limpia,
    
- clara,
    
- premium ligera,
    
- suficientemente profesional,
    
- no excesivamente disruptiva.
    

## Por qué

Porque todavía estamos validando comprensión.

Un diseño demasiado disruptivo puede distraer de la pregunta más importante:

**¿El usuario entiende y desea este producto?**

## Disrupción visual posterior

Una vez validado el flujo, se puede evolucionar hacia un diseño más memorable y diferenciado.

Pero la V1 debe priorizar claridad.

---

# Principios visuales no negociables

## 1. Claridad antes que belleza

Si algo se ve bonito pero no se entiende, se elimina.

## 2. Menor fricción visible

No mostrar pasos innecesarios.

## 3. Web/PWA con propósito

Cada vez que aparezca Web/PWA debe justificar su existencia.

## 4. WhatsApp natural

No convertir WhatsApp en un formulario disfrazado.

## 5. Confirmación humana

Crear, corregir y cerrar deben mostrar confirmación humana.

## 6. Privacidad implícita

No mostrar lectura de chats personales ni grupos.

## 7. Evidencia visible

La evidencia debe aparecer como parte central del valor.

## 8. Cierre contundente

El cierre debe sentirse como alivio operativo.
